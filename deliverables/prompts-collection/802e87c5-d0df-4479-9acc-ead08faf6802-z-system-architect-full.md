## Node.js REST API Architecture with Authentication

Here's the system architect's recommendation:

### Folder Structure

```
src/
├── config/           # Database, auth, app config
├── middleware/        # auth, error, validation, rateLimit
├── routes/           # Route definitions per resource
├── controllers/      # HTTP request/response handlers
├── services/         # Business logic layer
├── repositories/     # Data access layer
├── models/           # Data models (User, RefreshToken)
├── utils/            # JWT, password hashing, validators
└── app.js            # Express setup
```

### Key Architectural Decisions

| Decision | Choice | Why |
|----------|--------|-----|
| **Auth** | JWT + Refresh Tokens | Stateless, scalable, mobile-friendly |
| **Framework** | Express | Battle-tested, massive ecosystem |
| **DB** | PostgreSQL | ACID, relational, proven at scale |
| **Passwords** | bcrypt (12 rounds) | Industry standard, brute-force resistant |
| **Structure** | Controller → Service → Repository | Testable, clear separation of concerns |

### Authentication Flow

- **Access token**: Short-lived (15 min), stateless, not stored in DB
- **Refresh token**: Long-lived (7 days), hashed and stored in DB for revocation
- **Middleware**: `authenticate` (verify JWT), `authorize(...roles)` (role-based), `optionalAuth`

### Security Essentials (Day 1)

- **helmet** for security headers
- **cors** with explicit allowed origins
- **express-rate-limit** (stricter on `/auth/login` - 5 attempts per 15 min)
- **express-validator** for input validation
- Body size limits (`10kb`)
- Never log passwords or tokens

### Scalability Path

1. **Now (<1K req/min)**: Single server + PostgreSQL, no caching needed
2. **10x growth**: Add Redis (token blacklist, rate limiting), connection pooling, horizontal scaling behind load balancer
3. **100x growth**: Consider separating auth into its own service, read replicas, message queue for async ops

### Core Principle

**Start simple, scale when measured.** Don't add microservices, CQRS, or Kubernetes on day 1. This layered monolith will serve you well from 10 to 100K users with minimal refactoring.
