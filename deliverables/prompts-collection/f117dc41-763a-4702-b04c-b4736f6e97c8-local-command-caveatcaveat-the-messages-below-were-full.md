Yes, absolutely. You have several approaches:

1. **Canvas API** — Draw and animate on a `<canvas>` element using JavaScript. Best for pixel-level control, sprites, and complex rendering (e.g., platformers, shooters).

2. **CSS + DOM manipulation** — Use HTML elements styled with CSS and move them with JavaScript (`transform`, `position`). Works well for simpler games (e.g., Simon, card games, puzzle games).

3. **Game libraries** — Libraries like **Phaser**, **PixiJS**, or **Kaboom.js** abstract the Canvas API and provide physics, collision detection, sprite management, etc.

Your current Simon Game project is actually an example of approach #2 — a 2D game built with JavaScript (React) and CSS (Tailwind).

**Kaboom.js** is a lightweight JavaScript library for making 2D games quickly. Here's the overview:

## Core Concepts

- **Scene-based** — Organize game logic into scenes (menu, gameplay, game over)
- **Component system** — Game objects are built by composing components like `pos()`, `sprite()`, `area()`, `body()`
- **Built-in physics** — Gravity, collision detection, and platformer mechanics out of the box
- **Simple API** — Designed to be beginner-friendly with minimal boilerplate

## Key Features

- Sprite loading and animation
- Tilemaps and level design from ASCII art
- Input handling (keyboard, mouse, touch, gamepad)
- Audio playback
- Text rendering
- Tweens and timers
- Debug mode

## Basic Example

```js
import kaboom from "kaboom";

kaboom(); // initialize

loadSprite("bean", "sprites/bean.png");

const player = add([
  sprite("bean"),
  pos(80, 40),
  area(),
  body(),
]);

onKeyPress("space", () => {
  player.jump();
});
```

## Good to Know

- **Status**: The original Kaboom.js is no longer actively maintained. It was succeeded by **Kaplay** (a community fork that continues development).
- **Best for**: Game jams, prototypes, learning game dev, simple browser games
- **Not ideal for**: Large-scale or performance-intensive games (Phaser or PixiJS are better there)
- **No build tools required** — can run from a `<script>` tag, though it works with bundlers too

If you're considering it for a project, **Kaplay** (https://kaplayjs.com) is the recommended active alternative.
